# คู่มือการใช้งานระบบ RoPA

ระบบบันทึกกิจกรรมการประมวลผลข้อมูลส่วนบุคคล (Record of Processing Activities) ตาม พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล (PDPA)

คู่มือฉบับนี้สำหรับผู้ใช้งานทั่วไป (เจ้าหน้าที่หน่วยงาน) และผู้ดูแลระบบ (DPO/Admin)

---

## 1. แนะนำระบบ

ระบบ RoPA ใช้บันทึก "กิจกรรมการประมวลผลข้อมูลส่วนบุคคล" ของแต่ละหน่วยงาน เพื่อจัดทำบันทึกตามมาตรา 39 ของ PDPA โดย 1 รายการ = 1 กิจกรรม (เช่น การรับสมัครงาน การให้บริการลูกค้า)

ระบบรองรับ 4 บริษัทในเครือ: **BTSC, BID, EBM, NBM** โดยผู้ใช้แต่ละบริษัทจะเห็นเฉพาะข้อมูลของบริษัทตนเอง ผู้ดูแล (Admin) เห็นได้ทุกบริษัทและมีหน้า Dashboard สำหรับติดตามภาพรวม

**ผู้ใช้ 2 ประเภท:**
- **ผู้ใช้ทั่วไป (user):** สร้าง/แก้ไขรายการ RoPA ในขอบเขตหน่วยงานของตน
- **ผู้ดูแล (admin):** ตรวจสอบ/ตีกลับเอกสาร ดู Dashboard จัดการผู้ใช้ และออกรายงาน

---

## 2. การเข้าสู่ระบบ

1. เปิดเว็บไซต์ระบบ จะพบหน้าจอเข้าสู่ระบบ
2. กรอก **อีเมล** และ **รหัสผ่าน** แล้วกดเข้าสู่ระบบ
3. บริษัทของท่านถูกกำหนดจาก **โดเมนอีเมล** โดยอัตโนมัติ:

| โดเมนอีเมล | บริษัท |
|---|---|
| @bts.co.th | BTSC |
| @bid.com | BID |
| @ebm.com | EBM |
| @nbm.com | NBM |

> หากกรอกอีเมลหรือรหัสผ่านผิด ระบบจะแจ้ง "อีเมลหรือรหัสผ่านไม่ถูกต้อง"

**ออกจากระบบ:** กดปุ่ม "ออกจากระบบ" ที่มุมขวาบน (ในกล่องชื่อผู้ใช้)

---

## 3. ภาพรวมหน้าจอ

หลังเข้าสู่ระบบจะพบ **หน้ารายการ RoPA** เป็นหน้าหลัก ด้านบนมีแถบหัวเรื่องและข้อมูลผู้ใช้ (ชื่อ/ตำแหน่ง/ป้าย Admin ถ้ามี) ด้านล่างเป็นแถบเครื่องมือและตารางรายการ

---

## 4. หน้ารายการ RoPA

ตารางแสดงรายการทั้งหมดในขอบเขตที่ท่านมีสิทธิ์เห็น มี 8 คอลัมน์: ลำดับ · กิจกรรมการประมวลผล · ฝ่าย/ส่วน · บริษัท · ผู้บันทึก · ปรับปรุงล่าสุด · สถานะ · จัดการ

**สถานะ (badge):**
- 🟡 **ร่าง** — ยังกรอกไม่ครบ/ยังไม่บันทึกสมบูรณ์
- 🟢 **สมบูรณ์** — กรอกครบทุกขั้นแล้ว
- 🔴 **ตีกลับ** — ผู้ตรวจส่งกลับให้แก้ไข

### ความสามารถบนหน้ารายการ
- **ค้นหา:** พิมพ์ในช่องค้นหา (ค้นจาก กิจกรรม / ฝ่าย / ผู้บันทึก / บริษัท / แผนก)
- **เรียงลำดับ:** คลิกหัวคอลัมน์ คลิกซ้ำเพื่อสลับน้อย↔มาก (มีลูกศร ▲▼)
- **จัดกลุ่ม:** กดปุ่ม "📑 จัดกลุ่ม" เพื่อจัดเป็นลำดับชั้น บริษัท → ฝ่าย → ส่วน → แผนก (เริ่มแบบยุบ คลิกเพื่อกางแต่ละกลุ่ม)
- **ปรับความกว้างคอลัมน์:** ลากที่ขอบหัวคอลัมน์; กด "↔ รีเซ็ตคอลัมน์" เพื่อคืนค่าอัตโนมัติ
- **แบ่งหน้า:** เมื่อมีเกิน 50 รายการ ใช้ปุ่ม « แรก · ‹ ก่อนหน้า · ถัดไป › · สุด »

### ปุ่มแถบเครื่องมือ
| ปุ่ม | หน้าที่ |
|---|---|
| 📑 จัดกลุ่ม | สลับมุมมองจัดกลุ่ม |
| 💾 Save XML | บันทึกข้อมูลทั้งหมดเป็นไฟล์ XML |
| 📂 โหลด XML | นำเข้าข้อมูลจากไฟล์ XML |
| ⬇ JSON | ดาวน์โหลดข้อมูลเป็นไฟล์ JSON |
| 📊 Dashboard | เปิดแดชบอร์ด (เฉพาะ Admin) |
| 👥 จัดการผู้ใช้ | จัดการบัญชีผู้ใช้ (เฉพาะ Admin) |
| ⬇ Export Excel | ส่งออก Excel ตามรูปแบบ ROPA |
| 🧪 20/ฝ่าย-ส่วน | สร้างข้อมูลตัวอย่าง |
| 🗑 ล้างทั้งหมด | ลบรายการทั้งหมดในขอบเขตของท่าน |
| ＋ เพิ่มรายการใหม่ | สร้าง RoPA รายการใหม่ |

### การจัดการรายแถว
- **✎ แก้ไข** — เปิดรายการเพื่อแก้ไข
- **🗺️ Data Map** — ดูแผนผังการไหลข้อมูลของรายการ
- **⧉ ทำสำเนา** — คัดลอกเป็นรายการใหม่ (สถานะร่าง)
- **🗑 ลบ** — ลบรายการ (มีการยืนยัน)

## 5. การสร้างรายการ RoPA ใหม่

กดปุ่ม **＋ เพิ่มรายการใหม่**

### 5.1 กรอกข้อมูลผู้บันทึก
ระบุ **ชื่อ** และ **นามสกุล** (บังคับ) พร้อม ตำแหน่ง เบอร์ ฝ่าย ส่วน — เลือกฝ่ายก่อนจึงเลือกส่วนที่อยู่ในฝ่ายนั้น แล้วกด "ยืนยัน" เพื่อเข้าสู่แบบฟอร์ม 7 ขั้น

### 5.2 แบบฟอร์ม 7 ขั้นตอน
แถบซ้ายแสดง 7 ขั้น (มี ✓ เมื่อขั้นนั้นกรอกครบ) และแถบความคืบหน้า "กรอกครบ X%" คลิกที่ขั้นใดก็ได้เพื่อข้ามไปมา ช่องที่มีดอกจัน (*) คือบังคับ และถ้าเลือกตัวเลือก "อื่นๆ" จะมีช่อง "โปรดระบุ" ให้กรอกเพิ่ม

**ขั้นที่ 1 รายละเอียดทั่วไป:** บริษัทต้นสังกัด, ฝ่าย/ส่วน (1.1), กิจกรรมการประมวลผล (1.2), ผู้รับผิดชอบ (1.4), รูปแบบการบันทึก (1.5), ประเภทเจ้าของข้อมูล (1.6), บุคคลพิเศษ เช่น ผู้เยาว์ (1.7 — ถ้าเลือกไม่ใช่ "ไม่มี" ต้องระบุ Consent), ความถี่ (1.8)

**ขั้นที่ 2 การเก็บรวบรวม:** ข้อมูลทั่วไป (2.1) และ/หรือ ข้อมูลอ่อนไหว (2.2) อย่างน้อย 1 อย่าง, แหล่งที่มา (2.3), วัตถุประสงค์ (2.4), ฐานกฎหมาย (2.5), ฐานกฎหมายข้อมูลอ่อนไหว (2.6)

**ขั้นที่ 3 การใช้ภายใน:** เลือก "มีการแบ่งปัน/ไม่มี" หากมี ให้กด "➕ เพิ่มฝ่ายงานที่แบ่งปัน" แล้วกรอกแต่ละรายการ (ฝ่ายงาน, วัตถุประสงค์, ประเภทข้อมูล, ฐานกฎหมาย) — ฝ่ายงานห้ามซ้ำ

**ขั้นที่ 4 การเปิดเผยภายนอก:** เลือก "มีการเปิดเผย/ไม่มี" หากมี ให้เพิ่มผู้รับแต่ละราย (ผู้รับ, รายละเอียด, วัตถุประสงค์, มีสัญญา?, วิธีส่ง, มี DPA?) — แนบไฟล์ได้ ≤ 1MB — ผู้รับห้ามซ้ำ

**ขั้นที่ 5 การโอนต่างประเทศ:** เลือก "มี/ไม่มีการส่งออกนอกประเทศ" หากมี ต้องระบุ ประเทศปลายทาง, ข้อมูลบริษัทปลายทาง, วิธีการโอน, วัตถุประสงค์, และมาตรการคุ้มครอง (safeguard ตาม ม.28/29)

**ขั้นที่ 6 การเก็บรักษา:** เลือก "มี/ไม่มีการจัดเก็บ" หากมี ให้เพิ่มประเภทการเก็บรักษาแต่ละรายการ (ประเภท, เริ่มนับจาก, ระยะเวลา, มาตรการกายภาพ, มาตรการเทคนิค, แหล่งจัดเก็บ, วิธีลบ/ทำลาย) — ประเภทห้ามซ้ำ

**ขั้นที่ 7 สิทธิและการเข้าถึง:** บังคับกรอกเมื่อขั้นที่ 6 เลือก "มีการจัดเก็บ" — ระบุผู้มีสิทธิเข้าถึง (7.1), เงื่อนไข (7.2), วิธีการเข้าถึง (7.3)

### 5.3 บันทึกร่าง vs บันทึกสมบูรณ์
- **💾 บันทึกร่าง** — บันทึกได้ทุกเมื่อแม้ยังกรอกไม่ครบ (สถานะ "ร่าง")
- **✓ บันทึกรายการ** (ปุ่มขั้นสุดท้าย) — บันทึกเป็น "สมบูรณ์" ได้เมื่อกรอกครบทั้ง 7 ขั้นเท่านั้น หากยังไม่ครบ ระบบจะแจ้งขั้นที่ขาดและพาไปยังขั้นนั้น

## 6. การแก้ไข / ทำสำเนา / ลบ
- **แก้ไข:** กด ✎ ที่แถวรายการ
- **ทำสำเนา:** กด ⧉ จะได้รายการใหม่สถานะร่างที่คัดลอกข้อมูลมา
- **ลบ:** กด 🗑 และยืนยัน (ลบแล้วกู้คืนไม่ได้)

---

## 7. สถานะงานและการตรวจสอบ (สำหรับ Admin)

ผู้ดูแล (Admin) เปิดรายการแล้วจะมีปุ่ม **⛔ ตีกลับ (Reject)** ในแบบฟอร์ม
1. กด "ตีกลับ" แล้วกรอก **หมายเหตุ** (บังคับ) ว่าต้องแก้ไขจุดใด
2. กดยืนยัน รายการจะเปลี่ยนสถานะเป็น "ตีกลับ" และระบบแสดงข้อความยืนยัน (ในระบบสาธิตยังไม่ส่งอีเมลจริง)
3. เจ้าของรายการจะเห็นแบนเนอร์สีแดงพร้อมหมายเหตุเมื่อเปิดรายการ และต้องแก้ไขแล้วบันทึกใหม่

## 8. Data Map (แผนผังการไหลข้อมูล)

กด 🗺️ ที่รายการ หรือปุ่ม Data Map ในแบบฟอร์ม จะแสดงแผนผังการไหลข้อมูลของรายการนั้น (แหล่งที่มา → หน่วยงาน → รูปแบบบันทึก → การเก็บรักษา และการใช้ภายใน/เปิดเผย/โอนต่างประเทศ) หากข้อมูลยังไม่ครบจะมีแถบเตือนช่องที่ขาด

ปุ่มในหน้าต่างแผนผัง: **⬇ PNG**, **⬇ PDF**, **🖨 พิมพ์** (A4 แนวนอน), **✕ ปิด**

---

## 9. การนำเข้า / ส่งออกข้อมูล

- **ส่งออก XML / JSON:** ปุ่ม 💾 Save XML / ⬇ JSON บนแถบเครื่องมือ
- **นำเข้า XML:** ปุ่ม 📂 โหลด XML แล้วเลือกไฟล์ ระบบจะถามว่าจะ "รวมกับข้อมูลเดิม" หรือ "แทนที่ทั้งหมด"
- **ส่งออก Excel (รูปแบบ ROPA):** ปุ่ม ⬇ Export Excel เลือกบริษัทและฝ่าย/ส่วนที่ต้องการ แล้วกดดาวน์โหลด (แต่ละรายการ = 1 แถวในชีต ROPA)

---

## 10. Dashboard ผู้ดูแล (เฉพาะ Admin)

กดปุ่ม 📊 Dashboard เลือกกรองบริษัทได้ที่มุมขวาบน ประกอบด้วย:

- **🔔 แถบแจ้งเตือน** — รายการความเสี่ยง/งานค้างที่ต้องจัดการเร่งด่วน (สีแดง=ร้ายแรง, ส้ม=เฝ้าระวัง)
- **โซนภาพรวม (Scorecard)** — %สมบูรณ์รวม, ความเสี่ยงร้ายแรง, ปิดงานแต่ยังไม่ compliant, งานค้าง>30วัน, รอตรวจ/ตีกลับ + กราฟแนวโน้ม %สมบูรณ์
- **โซนความเสี่ยง & Compliance** — แยกตามชนิด (อ่อนไหวไร้ฐานกฎหมาย ม.26, โอนตปท.ไร้ safeguard ม.28/29, เปิดเผยขาด DPA, ไม่มีกำหนดลบ, บุคคลพิเศษ Consent=N), heatmap รายบริษัท และ **Remediation Worklist** ที่กดเปิดแก้ไขรายการได้และส่งออกได้
- **โซนความครอบคลุม (Coverage)** — หน่วยงานที่ยังไม่มี RoPA (🔴), มีแต่ร่าง (🟡), สมบูรณ์ (🟢) และกิจกรรมที่ยังไม่ครอบคลุม
- **โซนคุณภาพข้อมูล** — ช่องที่มักกรอกไม่ครบ และ heatmap ฝ่าย×ขั้นตอน
- **โซนติดตามงาน** — งานค้าง 7-14/15-30/>30 วัน, ถูกตีกลับ, เลยกำหนด/ใกล้ครบกำหนด, กิจกรรมล่าสุด, ฝ่ายที่ต้องเร่ง

**รายงานผู้บริหาร:** ปุ่ม 📄 รายวัน / รายสัปดาห์ / รายเดือน ดาวน์โหลดเป็นไฟล์ Excel หลายชีต (สรุปผู้บริหาร, รายบริษัท, แจ้งเตือน, Worklist, หน่วยงานจุดบอด)

**📌 Snapshot:** กดเพื่อบันทึกภาพรวมของวันนี้เข้าประวัติ (กราฟแนวโน้มจะค่อย ๆ มีเส้นยาวขึ้น ระบบยังเก็บอัตโนมัติทุกวันด้วย)

ส่วนที่ยาวมาก (Worklist, รายการหน่วยงาน, กิจกรรมล่าสุด ฯลฯ) มีการ **แบ่งหน้า** ให้กดเปลี่ยนหน้าได้

## 11. การจัดการผู้ใช้ (เฉพาะ Admin)

กดปุ่ม 👥 จัดการผู้ใช้
- **เพิ่มผู้ใช้:** กรอกอีเมล (กำหนดบริษัทจากโดเมน), บทบาท (user/admin), ชื่อ, ตำแหน่ง, ฝ่าย, ส่วน, แผนก, รหัสผ่าน
- **แก้ไข:** เว้นช่องรหัสผ่านว่างไว้หากไม่ต้องการเปลี่ยน
- **ลบ:** ลบบัญชีตัวเองไม่ได้ และต้องมีผู้ใช้เหลืออย่างน้อย 1 บัญชี

> ลำดับชั้นการมองเห็น: ผู้ใช้ระดับ "ฝ่าย" เห็นทั้งฝ่าย, ระดับ "ส่วน" เห็นทั้งส่วน, ถ้าระบุ "แผนก" จะเห็นเฉพาะข้อมูลของตัวเอง — ทั้งหมดจำกัดอยู่ในบริษัทของผู้ใช้

## 12. คำถามที่พบบ่อย (FAQ)

**ถาม: ทำไมเห็นเฉพาะข้อมูลบางส่วน?**
ตอบ: ระบบจำกัดการมองเห็นตามบริษัท (โดเมนอีเมล) และตามลำดับชั้นหน่วยงานของท่าน ผู้ดูแล (Admin) เท่านั้นที่เห็นทุกบริษัท

**ถาม: กดบันทึกสมบูรณ์ไม่ได้?**
ตอบ: ต้องกรอกครบทั้ง 7 ขั้นก่อน ระบบจะแจ้งขั้นที่ยังขาด ระหว่างนั้นใช้ "บันทึกร่าง" เก็บงานไว้ก่อนได้

**ถาม: เลือก "อื่นๆ" แล้วต้องทำอะไร?**
ตอบ: จะมีช่อง "โปรดระบุ" โผล่ขึ้นและเป็นช่องบังคับ ต้องกรอกรายละเอียดเพิ่ม

**ถาม: รายการถูกตีกลับ ต้องทำอย่างไร?**
ตอบ: เปิดรายการ อ่านหมายเหตุจากผู้ตรวจในแบนเนอร์สีแดง แก้ไขให้ครบแล้วบันทึกใหม่

**ถาม: ความเสี่ยง "ร้ายแรง" บน Dashboard คืออะไร?**
ตอบ: คือรายการที่ขัดต่อ PDPA โดยตรง เช่น ใช้ข้อมูลอ่อนไหวโดยไม่มีฐานกฎหมาย (ม.26), โอนต่างประเทศโดยไม่มีมาตรการคุ้มครอง (ม.28/29), หรือเปิดเผยให้บุคคลภายนอกโดยไม่มี DPA — ควรเร่งแก้ไขก่อน
