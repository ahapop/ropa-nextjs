# คู่มือการใช้งานระบบ RoPA ฉบับเต็ม (พร้อมภาพประกอบ)

ระบบบันทึกกิจกรรมการประมวลผลข้อมูลส่วนบุคคล (Record of Processing Activities) ตาม พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA)

คู่มือนี้อธิบายทุกฟังก์ชันของระบบทีละขั้นตอนพร้อมภาพหน้าจอจริง สำหรับผู้ใช้งานทั่วไป (เจ้าหน้าที่หน่วยงาน) และผู้ดูแลระบบ (DPO/Admin)

---

## สารบัญ
1. แนะนำระบบและบทบาทผู้ใช้
2. การเข้าสู่ระบบ
3. หน้ารายการ RoPA
4. การค้นหาและการจัดกลุ่ม
5. การสร้างรายการ RoPA ใหม่ (ผู้บันทึก + 7 ขั้นตอน)
6. การบันทึกร่าง / บันทึกสมบูรณ์
7. การตรวจสอบและตีกลับเอกสาร (Admin)
8. แผนผังการไหลข้อมูล (Data Map)
9. การนำเข้า/ส่งออกข้อมูล (Excel / XML / JSON)
10. แดชบอร์ดผู้ดูแล (Dashboard)
11. รายงานผู้บริหาร (Daily/Weekly/Monthly)
12. การจัดการผู้ใช้
13. คำถามที่พบบ่อย (FAQ)

## 1. แนะนำระบบและบทบาทผู้ใช้

ระบบ RoPA ใช้บันทึก "กิจกรรมการประมวลผลข้อมูลส่วนบุคคล" ของแต่ละหน่วยงาน โดย **1 รายการ = 1 กิจกรรม** (เช่น การรับสมัครงาน การให้บริการลูกค้า) เพื่อจัดทำบันทึกตามมาตรา 39 แห่ง PDPA

ระบบรองรับ **4 บริษัทในเครือ**: BTSC, BID, EBM, NBM โดยผู้ใช้แต่ละบริษัทจะเห็นเฉพาะข้อมูลของบริษัทตนเอง

| บทบาท | สิทธิ์การใช้งาน |
|---|---|
| ผู้ใช้ทั่วไป (user) | สร้าง/แก้ไขรายการ RoPA ในขอบเขตหน่วยงานของตน (ตามลำดับชั้น ฝ่าย/ส่วน/แผนก) |
| ผู้ดูแล (admin / DPO) | เห็นทุกบริษัท ตรวจสอบ/ตีกลับเอกสาร เข้าแดชบอร์ด จัดการผู้ใช้ ออกรายงาน |

## 3. หน้ารายการ RoPA

หลังเข้าสู่ระบบจะพบหน้ารายการ RoPA เป็นหน้าหลัก แสดงรายการทั้งหมดในขอบเขตที่ท่านมีสิทธิ์เห็น

![หน้ารายการ RoPA พร้อมแถบเครื่องมือและตาราง](docs/img/02-list.png)

**คอลัมน์ในตาราง:** ลำดับ · กิจกรรมการประมวลผล · ฝ่าย/ส่วน · บริษัท · ผู้บันทึก · ปรับปรุงล่าสุด · สถานะ · จัดการ

**สถานะ (ป้ายสี):**
- 🟡 ร่าง — ยังกรอกไม่ครบ/ยังไม่บันทึกสมบูรณ์
- 🟢 สมบูรณ์ — กรอกครบทุกขั้นแล้ว
- 🔴 ตีกลับ — ผู้ตรวจส่งกลับให้แก้ไข

**ปุ่มแถบเครื่องมือ (ซ้าย→ขวา):**
| ปุ่ม | หน้าที่ |
|---|---|
| 📑 จัดกลุ่ม | สลับมุมมองจัดกลุ่มตามหน่วยงาน |
| 💾 Save XML | บันทึกข้อมูลทั้งหมดเป็นไฟล์ XML |
| 📂 โหลด XML | นำเข้าข้อมูลจากไฟล์ XML |
| ⬇ JSON | ดาวน์โหลดข้อมูลเป็นไฟล์ JSON |
| 📊 Dashboard | เปิดแดชบอร์ด (เฉพาะ Admin) |
| 👥 จัดการผู้ใช้ | จัดการบัญชีผู้ใช้ (เฉพาะ Admin) |
| ⬇ Export Excel | ส่งออก Excel ตามรูปแบบ ROPA |
| 🧪 20/ฝ่าย-ส่วน | สร้างข้อมูลตัวอย่าง |
| 🗑 ล้างทั้งหมด | ลบรายการทั้งหมดในขอบเขตของท่าน |
| ＋ เพิ่มรายการใหม่ | สร้าง RoPA รายการใหม่ |

**ปุ่มจัดการรายแถว:** ✎ แก้ไข · 🗺️ Data Map · ⧉ ทำสำเนา · 🗑 ลบ

**การเรียงลำดับ:** คลิกหัวคอลัมน์เพื่อเรียง คลิกซ้ำเพื่อสลับทิศ (มีลูกศร ▲▼) · **ปรับความกว้างคอลัมน์:** ลากที่ขอบหัวคอลัมน์ · **แบ่งหน้า:** เมื่อเกิน 50 รายการจะมีปุ่มเปลี่ยนหน้าด้านล่าง

## 5. การสร้างรายการ RoPA ใหม่

กดปุ่ม **＋ เพิ่มรายการใหม่** ที่หน้ารายการ

### 5.1 กรอกข้อมูลผู้บันทึก
ระบบจะให้กรอกข้อมูลผู้บันทึกก่อน — บังคับ **ชื่อ** และ **นามสกุล** ส่วน ตำแหน่ง/เบอร์/ฝ่าย/ส่วน กรอกเพิ่มได้ (เลือกฝ่ายก่อนจึงเลือกส่วนในฝ่ายนั้น) แล้วกด **ยืนยัน**

![กล่องกรอกข้อมูลผู้บันทึก](docs/img/05-recorder.png)

### 5.2 แบบฟอร์ม 7 ขั้นตอน — ภาพรวม
แถบซ้ายแสดง 7 ขั้น (มีเครื่องหมาย ✓ เมื่อขั้นนั้นกรอกครบ) และแถบความคืบหน้า "กรอกครบ X%" · คลิกที่ขั้นใดก็ได้เพื่อข้ามไปมา · ช่องที่มีดอกจัน (*) คือบังคับ · หากเลือกตัวเลือก "อื่นๆ" จะมีช่อง "โปรดระบุ" ให้กรอกเพิ่ม (บังคับ)

### ขั้นที่ 1 — รายละเอียดทั่วไป

## 6. การบันทึกร่าง / บันทึกสมบูรณ์
- **💾 บันทึกร่าง** — บันทึกได้ทุกเมื่อแม้ยังกรอกไม่ครบ (สถานะ "ร่าง")
- **✓ บันทึกรายการ** (ปุ่มขั้นสุดท้าย) — บันทึกเป็น "สมบูรณ์" ได้เมื่อกรอกครบทั้ง 7 ขั้นเท่านั้น หากยังไม่ครบ ระบบจะแจ้งขั้นที่ขาดและพาไปยังขั้นนั้น

## 7. การตรวจสอบและตีกลับเอกสาร (เฉพาะ Admin)

ผู้ดูแล (Admin) เปิดรายการแล้วจะเห็นปุ่ม **⛔ Reject (ตีกลับ)** ในแถบล่างของแบบฟอร์ม กดเพื่อส่งกลับให้เจ้าของแก้ไข โดย **ต้องกรอกหมายเหตุ** ระบุจุดที่ต้องแก้

![กล่องตีกลับเอกสารพร้อมหมายเหตุ](docs/img/24-reject.png)

เมื่อตีกลับ รายการจะเปลี่ยนสถานะเป็น "ตีกลับ" และเจ้าของจะเห็นแบนเนอร์สีแดงพร้อมหมายเหตุเมื่อเปิดรายการ ต้องแก้ไขแล้วบันทึกใหม่

## 8. แผนผังการไหลข้อมูล (Data Map)

ปุ่มในหน้าต่าง: **⬇ PNG**, **⬇ PDF**, **🖨 พิมพ์** (A4 แนวนอน), **✕ ปิด** · หากข้อมูลยังไม่ครบจะมีแถบเตือนช่องที่ขาด

## 9. การนำเข้า/ส่งออกข้อมูล

### 9.1 ส่งออก Excel (รูปแบบ ROPA)
กดปุ่ม **⬇ Export Excel** เลือกบริษัทและฝ่าย/ส่วนที่ต้องการ ระบบแสดงจำนวนรายการที่จะส่งออก แล้วกดดาวน์โหลด (แต่ละรายการ = 1 แถวในชีต ROPA)

![กล่องส่งออก Excel](docs/img/17-excel.png)

### 9.2 ส่งออก/นำเข้า XML และ JSON
- **💾 Save XML** — บันทึกข้อมูลทั้งหมดเป็นไฟล์ XML
- **📂 โหลด XML** — นำเข้าจากไฟล์ XML โดยเลือก "รวมกับข้อมูลเดิม" หรือ "แทนที่ทั้งหมด"
- **⬇ JSON** — ดาวน์โหลดข้อมูลเป็นไฟล์ JSON

## 10. แดชบอร์ดผู้ดูแล (Dashboard) — เฉพาะ Admin

กดปุ่ม **📊 Dashboard** กรองบริษัทได้ที่มุมขวาบน

### 10.1 ภาพรวมผู้บริหาร + แจ้งเตือน + แนวโน้ม
ด้านบนเป็นแถบ **แจ้งเตือน** (สีแดง=ร้ายแรง, ส้ม=เฝ้าระวัง) ตามด้วยการ์ด KPI (%สมบูรณ์, เสี่ยงร้ายแรง, ปิดงานแต่ยังไม่ compliant, งานค้าง>30วัน, รอตรวจ/ตีกลับ) และกราฟแนวโน้ม %สมบูรณ์

![แดชบอร์ด: ภาพรวม + แจ้งเตือน + แนวโน้ม](docs/img/18-dashboard-top.png)

## 12. การจัดการผู้ใช้ (เฉพาะ Admin)

กดปุ่ม **👥 จัดการผู้ใช้** เพื่อเพิ่ม/แก้ไข/ลบบัญชี

![หน้าจัดการผู้ใช้](docs/img/23-users.png)

- **เพิ่มผู้ใช้:** กรอกอีเมล (กำหนดบริษัทจากโดเมน), บทบาท (user/admin), ชื่อ, ตำแหน่ง, ฝ่าย, ส่วน, แผนก, รหัสผ่าน
- **แก้ไข:** เว้นช่องรหัสผ่านว่างไว้หากไม่ต้องการเปลี่ยน
- **ลบ:** ลบบัญชีตัวเองไม่ได้ และต้องมีผู้ใช้เหลืออย่างน้อย 1 บัญชี

> ลำดับชั้นการมองเห็น: ระดับ "ฝ่าย" เห็นทั้งฝ่าย, ระดับ "ส่วน" เห็นทั้งส่วน, ถ้าระบุ "แผนก" เห็นเฉพาะข้อมูลของตัวเอง — ทั้งหมดจำกัดอยู่ในบริษัทของผู้ใช้

## 13. คำถามที่พบบ่อย (FAQ)

**ถาม: ทำไมเห็นเฉพาะข้อมูลบางส่วน?**
ตอบ: ระบบจำกัดการมองเห็นตามบริษัท (โดเมนอีเมล) และลำดับชั้นหน่วยงาน ผู้ดูแล (Admin) เท่านั้นที่เห็นทุกบริษัท

**ถาม: กดบันทึกสมบูรณ์ไม่ได้?**
ตอบ: ต้องกรอกครบทั้ง 7 ขั้นก่อน ระบบจะแจ้งขั้นที่ขาด ระหว่างนั้นใช้ "บันทึกร่าง" เก็บงานไว้ก่อนได้

**ถาม: เลือก "อื่นๆ" แล้วต้องทำอะไร?**
ตอบ: จะมีช่อง "โปรดระบุ" โผล่ขึ้นและเป็นช่องบังคับ ต้องกรอกรายละเอียดเพิ่ม

**ถาม: รายการถูกตีกลับ ต้องทำอย่างไร?**
ตอบ: เปิดรายการ อ่านหมายเหตุจากผู้ตรวจในแบนเนอร์สีแดง แก้ไขให้ครบแล้วบันทึกใหม่

**ถาม: ความเสี่ยง "ร้ายแรง (Critical)" บนแดชบอร์ดคืออะไร?**
ตอบ: รายการที่ขัดต่อ PDPA โดยตรง เช่น ใช้ข้อมูลอ่อนไหวโดยไม่มีฐานกฎหมาย (ม.26), โอนต่างประเทศโดยไม่มีมาตรการคุ้มครอง (ม.28/29), หรือเปิดเผยให้บุคคลภายนอกโดยไม่มี DPA — ควรเร่งแก้ไขก่อน โดยใช้ Remediation Worklist กดเข้าไปแก้ได้ทันที
